General
Off - don't use Camilla DSP
Custom - custom alsa_cdsp configuration is used, don't touch any alsa_cdsp or camilla configuration.
Else use the selected Camilla DSP configuration file. This one will be assigned to alsa_cdsp.
$_select[cdsp_patch_playback_device1] $_select[cdsp_patch_playback_device0]
Automatic set correct playback device and format for the CamillaDSP configuration.
Disable if you want to used an other play back device then the default of moOde,
for example if you want to create a multi channel crossover.
Remove: delete the selected configuation file.
Download: download the selected configuation file.
Upload new: upload a configuration file, existing ones are overwritten.
Check: check if the selected configuration is a valid CamillaDSP configuration.

Changes to the configuration are effective immediately after pausing/stopping playback.
Information on how to create CamillaDSP configuration files can be found on the CamillaDSP homepage.

If CamillaDSP can't play audio, errors will be reported to /var/log/mpd/log
In case this is due to channel counts or sampling rates which are not supported by your DAC,
you can either handle this in CamillaDSP with mixers or resamplers
or change the formats which are passed to CamillaDSP in /etc/alsa/conf.d/camilladsp.conf
If this file is erroneous, /var/log/moode.log and /var/log/syslog may also provide hints.
With the currently selected sound device, this is the minimal config, that you need to get any sound through CamillaDSP:
devices: samplerate: 44100 # irrelevant if resampling is disabled - then it will be overridden with the incoming sample rate chunksize: 1024 queuelimit: 2 # increasing this introduces significant latency when stopping playback or changing songs # e.g.: samplerate=44100, chunksize=1024, queuelimit=44 => latency ~ 1s capture: type: Stdin channels: 2 # irrelevant - will always be overridden (currently always 2) format: S32LE # irrelevant - will always be overridden with the incoming sample format playback: type: Alsa channels: 2 device: "$sound_device_type:$current_sound_device_number" # by default, this is automatically updated when configuration is choosen. (correct value is only shown, when playback is stopped.) # format: $sound_device_sample_format # supported formats are: $sound_device_supported_sample_formats # correct value is only shown, when playback is stopped

Any reference to a convolution file should be relative like ../coeffs/yourfile.wav
